# design-sync notes — @philemon/ui

Repo-specific gotchas for future syncs. Read this before re-running.

## How this DS builds (package shape, source-only)

- `@philemon/ui` ships **source TypeScript** (`main`/`types` → `./src/index.ts`),
  no `dist/`, no build script. The converter bundles directly from the source
  barrel.
- **You MUST pass `--entry ./packages/ui/src/index.ts`.** Without it the build
  derives `PKG_DIR` as `<node-modules>/@philemon/ui`, which doesn't exist (pnpm
  doesn't self-install the workspace package) and crashes in `projectFor`. With
  `--entry`, PKG_DIR walks up to `packages/ui` correctly, and ts-morph extracts
  components + props straight from source (entry resolves to `src/index.ts`, so
  this is NOT synth mode — props are full-fidelity).
- `--node-modules packages/ui/node_modules` (react/react-dom resolve there;
  `@philemon/types` is a type-only import in Badge, erased at bundle).
- Build command:
  `node .ds-sync/package-build.mjs --config .design-sync/config.json --node-modules packages/ui/node_modules --entry ./packages/ui/src/index.ts --out ./ds-bundle`

## CSS (the one non-obvious piece)

- `cfg.cssEntry = dist/philemon-ui.css` is a **generated, gitignored** file, not a
  repo source file. `cfg.buildCmd = node .design-sync/build-css.mjs` regenerates it
  every sync — **always run buildCmd (or the driver, which doesn't; run buildCmd
  manually) before the converter.**
- Why generated: the package's real stylesheet is `src/global.css`, which uses
  relative `@import "./tokens.css"` / `"./components.css"`. The converter copies
  `cssEntry` verbatim into `_ds_bundle.css` at the bundle ROOT, where those
  relative imports would 404 → every design renders unstyled. `build-css.mjs`
  inlines tokens + components + base into one self-contained file.
- It also prepends the **Geist Mono** Google-Fonts `@import` (same family/weights
  the host app loads in `apps/furnishing-planner/web/index.html`). This is why
  validate reports `[FONT_REMOTE]` for "Geist Mono" — expected, informational, and
  the correct behavior (designs built with the DS load the real brand font at
  runtime). If the host ever changes the font/weights, update the URL in
  `build-css.mjs`.

## Grouping

- All 20 components land under one `general` group: the source dir is
  `src/components/` (a generic name the grouper skips), and there are no docs with
  `category` frontmatter. Acceptable but flat. To improve later: add `@category`
  JSDoc to each component in source (also enriches `prompt.md`), or `docsMap` stubs
  (but a stub `.md` REPLACES the synthesized prompt.md with just frontmatter —
  empties the prompt body, so don't).

## Overrides

- `Table` and `TableWrap` use `cardMode: "column"` (full card width, one export per
  row) — they're wider than a grid cell. Set defensively (render check was skipped,
  so `[GRID_OVERFLOW]` never ran to confirm).

## Known render warns (check re-sync warns against this list)

- `[FONT_REMOTE] "Geist Mono"` — remote font @import, by design.
- `[RENDER_SKIPPED]` — only when built with `--no-render-check` (see below).

## Re-sync risks

- **Previews were NEVER visually verified.** This run was done with
  `--no-render-check` (no Playwright on the machine; user opted out). All 20
  authored previews compiled and reference `window.PhilemonUI` correctly, but none
  were screenshot-graded. A future sync on a machine with Playwright + Chromium
  should run the full render check / `package-capture.mjs` grading and fix any
  composition that renders wrong. Until then, treat preview correctness as
  best-effort (authored carefully from the real source, but unconfirmed).
- **Dark-only DS.** Components render white-on-black. Authored previews each wrap
  content in a local `Frame` that paints `#000` because the preview harness body is
  white — if you re-author, keep that, or cards look blank/invisible.
- `conventions.md` enumerates token/component names; all were validated against
  this build's artifacts. Re-validate on re-sync (the conventions step does this).
- `dist/philemon-ui.css` is gitignored and regenerated by `build-css.mjs` — a fresh
  clone has no CSS until buildCmd runs.
- Component count must stay 20 (PascalCase value exports from `src/index.ts`). A new
  export shows up automatically; `cx`/`formatEuro`/`numCell` are correctly excluded
  (not PascalCase).
